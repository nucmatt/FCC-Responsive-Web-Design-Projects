Introduction

As we have learned, HTML contains the content of your web page and is used to describe the structure of this content. Some of your content may well be complex data sets that you would like your users to see. Making sense of complex data is aided by presenting it in a logical, hierarchical structure. This is where HTML tables come in. HTML tables give you a way to present your complex data in tabular form. These tables can contain not only your data but also headers, footers, captions, and even comments to help your users understand the data you are presenting.

Building a HTML Table

To build an HTML table we should first understand the basic structure of any good table. In essence every good table is a grid made up of columns and rows. The intersetion of a column and a row is known as a table cell. Each table cell contains a piece of data the table is presenting. With proper headings, a summation of the data presented, and clarifying comments added where necessary even complex data can becomes readily understandable.

Knowing the basic structure of any good table we can now see how to construct a basic table in HTML. HTML tables are constructed by nesting a variety of elements between table tags. HTML organizes tables into rows (vice columns) using the table row tr element. Each table row is made up of one or more table data td elements. Columns are formed automatically by aligning table data elements in subsequent table rows vertically. In general, the first row of the table is made up of table heading th elements to organize and give meaning to the subsequent rows of data. Table heading text is bold by default. You can override this and apply your own styling using CSS.

To summarize, every HTML table is made up of four essential elements. These are:

  • table tags - These contain all the elements that make an HTML table.
  • tr - HTML tables are organized into rows using table row elements.
  • td - Each piece of table data is contained within table data elements.
  • th - The first table row is made up of table headings to give meaning to the table data.

Table Cells

From the discussion of the basics of any table and how to build an HTML table, we can now surmise that table cells are created with table header and table data elements. But what if our table is more complex still and certain parts need to span multiple rows or columns? Luckily, there are two attributes that can be added to the elements that make up our HTML table.

First, let us tackle the issue of table cells that we would like to span multiple rows or columns. To create a table cell that spans multiple columns you will need to add the column span colspan="x" attribute to a table data or table element header element. The "x" value must be an integer and is set to "1" by default. To ensure that your table columns continue to align properly, it is important that all of the column span attributes for each table row are equal.

Table cells that span multiple rows is similar to how table cells span multiple columns. You will add the row span rowspan="x" attribute to the table row element. The "x" is again an integer and is set to 1 by default. To properly align your table data with a row spanning multiple rows, reduce the number of table data entries in "x-1" subsequent table row elements by one.

Add Structure to a HTML Table

While learning to construct an HTML document you have learned that the semantic structure of your document is as important as the content itself. Having a semantically organized HTML document aids you in organizing your content, aids anyone who may need to modify or update your document in the future, and aids search engines in categorizing and finding your content. An HTML table is an excellent way to present complex data for analysis and will benefit from appropriate semantic structure as well. To this end HTML tables have three special elements that semantically structure your table. These elements are:

  • The table head thead element groups the header content of an HTML table. It must be a direct descendent of the table element and comes before any table row elements.
  • The table body tbody element contains the content data your table is presenting, much as the body element contains the content of an HTML document. It must be a direct descendent of the table element and is placed after the table header element.
  • The table footer tfoot element contains the summary content of the tabular data presented in the table body. It must the a direct descendent of the table element and come after the table header element. The table footer element may be placed before or after the table body element in the HTML but will always be displayed to the user after the table body content.
Caption and Comment a HTML Table

Now that we can construct a proper HTML table and organize it well we can examine additional ways to aid our users in understanding the data presented. One item our table lacks is a caption describing the contents of the table. This can easily be accomplished with a header element. However, our goal is to aid all users, including those using assistive technologies. Luckily HTML tables have the caption element we can insert within the table tags. The caption element is inserted as the first descendent of the parent table element.

Your table data may come from an external source that needs attribution or you may want to provide context for your table data with explanations. Since there is currently no special element we can use to accomplish this commentary we will need to be creative. Within the table footer element we can add an additional table row element that spans the the total width of the table using the colspan attribute.

Styling a HTML Table

Styling HTML tables used to be a tedious process since most styling was done within the table itself. Thankfully, with the advent of HTML5, most of that styling is deprecated and is now done easily using CSS. General CSS is beyond the scope of this tutorial but there are a few special CSS rules regarding HTML tables that will be covered here.

The special CSS rules for HTML tables apply when borders are added to your table. Borderless table cells can get confusing quickly so it is generally best to add borders to both the table and the table cells. Adding borders to your table is done by adding the border property to your table, table data, and table header elements. Adding a border to your table element behaves exactly as adding a border to any other block element. However, when you add borders to your table header and table data elements, each individual cell received its own border and the table cells are separated margins on all four sides. Thia may be what you want and you simply want to to control the margins around the cells or you may want to collapse the individual borders for a cleaner look. This is where the following two CSS rules come in:

  • The border-spacing: x y; CSS rule is used to set the margins between cell borders. The 'x' value sets the horizontal margins and the 'y' sets the vertical margins. Setting only one value for the rule sets the margin for all sides. The values can be any valid CSS length units.
  • The border-collapse: collapse; CSS rule collapses all the table cells borders into a single border. This will overried the border-spacing rule if it is set.

Here is our example table with just these few lines of CSS added:


<style>
 table {
  border-collapse: collapse;}
 td, th {
  border: 1px, solid #CCC;}
</style>

HTML Tables vs CSS Tables

In the early days of website design it was common to use HTML tables as a layout device. With the advent of CSS and modern browsers using agreed upon standards this layout is now obsolete. So why do HTML tables exist at all anymore? Wouldn't it be simpler to use CSS tables for everything? Simply put, HTML tables are particularly good at creating a logical, heirarchical structure to present complex data sets. CSS tables are a presentation device to position lists or other content next to each other in columns. To summarize, use CSS tables to present content to your users in columns and use HTML tables when you have complex data sets that needs a strong, logical structure to be useful.
